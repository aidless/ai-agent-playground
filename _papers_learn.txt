Q: What concrete techniques do Reflexion, ReAct, and Toolformer propose for agent self-improvement? Extract specific patterns we can implement.
Based solely on the provided research papers, the concrete techniques for agent self-improvement proposed by **Reflexion**, **ReAct**, and **Toolformer** are as follows:

### 1. Reflexion (Paper [3])
**Technique: Verbal Reinforcement Learning via Self-Reflection**
- **Concrete Pattern:** The agent generates a textual summary of its own past actions, failures, and successes (a "verbal reflection") and stores this in an episodic memory buffer. On subsequent attempts, the agent retrieves and conditions its next actions on these reflections, effectively learning from trial-and-error without updating model weights.
- **Implementation Pattern:** 
  1. **Act:** Execute a task in the environment.
  2. **Observe:** Collect feedback (e.g., success/failure signal, error messages).
  3. **Reflect:** Prompt the LLM to produce a natural language critique (e.g., "I failed because I used the wrong API call; next time I should try X").
  4. **Store:** Save the reflection in a memory buffer.
  5. **Reuse:** On the next task attempt, inject the relevant reflection into the prompt as context.

### 2. ReAct (Paper [2] ¨C referenced as a component of agentic systems)
**Technique: Interleaved Reasoning and Acting with Self-Correction**
- **Concrete Pattern:** The agent alternates between generating reasoning traces (e.g., "I need to find the current weather") and taking actions (e.g., calling a weather API). The reasoning trace serves as a form of self-monitoring, allowing the agent to detect errors mid-execution and adjust its plan.
- **Implementation Pattern:**
  1. **Thought:** Generate a natural language reasoning step (e.g., "To answer this, I first need to check the user's location").
  2. **Action:** Execute a tool call or environment interaction.
  3. **Observation:** Receive the result.
  4. **Loop:** Repeat, using the observation to inform the next thought. If the observation contradicts the plan, the agent can self-correct by generating a new thought (e.g., "The location API failed, so I'll ask the user directly").

### 3. Toolformer (Paper [2] ¨C referenced as a component of agentic systems)
**Technique: Self-Supervised Tool Learning via API Calls**
- **Concrete Pattern:** The agent learns to autonomously decide when and how to call external tools (e.g., calculators, search engines, code interpreters) by generating API calls in its output text. The model is fine-tuned on examples where it predicts the utility of a tool call based on the context.
- **Implementation Pattern:**
  1. **Annotate:** For a given prompt, generate candidate API calls (e.g., `[Calculator(3+5)]`).
  2. **Execute:** Run the API call and get the result.
  3. **Filter:** Keep only those calls where the API result improves the final answer (e.g., reduces perplexity or increases correctness).
  4. **Fine-tune:** Train the model on the filtered examples so it learns to insert appropriate tool calls autonomously during inference.
